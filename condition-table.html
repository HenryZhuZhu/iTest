Condition Table

Plan and version every test condition in one shared editor — replacing scattered spreadsheets with an online source of truth.

Capabilities

Everything that used to live in a spreadsheet, online.

Edit in the browser.

Inline cell editing, multi-select, paste from Excel — everything you'd expect from a desktop spreadsheet, with none of the version-naming dance that follows.

Versioned by default.

Every save is a revision. Diff snapshots, restore in one click.

Built for teams.

Threads, change requests, permissions tied to product line.

Validated before publish.

Schema checks, unit consistency, ranges, and dependencies are enforced at save time — broken tables never reach the floor.

Built for

PTE engineers, test program owners, and the product leads who review their work — across DRAM and emerging-memory product lines.

Condition Table is the canonical source of truth for every parameter that drives a test run. Anything downstream — Test Methods, Test Management, even DFT vectors — reads from here. When it's one click and one history, the rest of the workflow gets simpler.
